Aide pour l'ajout de documents d'information

Retour à la page d'aministration des documents d'information

Il est possible d'ajouter sur le serveur Papillon des documents monolingues ou multilingues de n'importe quel type. Les fichiers de type archive (TAR ou ZIP) peuvent être multilingues, tandis que les fichiers simples (HTML, PDF, RTF, GIF, etc.) sont obligatoirement monolingues.
Un fichier compréssé avec Gzip est d'abord décompacté; le fichier ainsi obtenu n'est pas ajouté directement, mais traité exactement comme si il avait été envoyé sous forme non compréssée.

1. Ajout d'archives (.tar, .zip, .tar.gz)

1.1. Processus

L'archive est décompactée et son contenu est analysé, puis ajouté. Il peut s'agir d'un unique répertoire archivé, auquel cas le terme "racine" désigne le contenu de ce répertoire. Il est possible de placer à la racine de l'archive un fichier au format XML nommé info_doc.xml qui remplace la saisie des informations du formulaire d'ajout de documents. Ce fichier de decription info_doc.xml de l'archive précise :

  • la section dans laquelle vous souhaitez placer le document,
  • le nom du ou des auteurs du document,
  • le titre du document dans chacune des langues disponibles,
  • la date de création du document (optionnel),
  • des réféfrences si c'est une publication (optionnel).

Si des informations ont été entrées dans le formulaire, elles sont ignorées et remplacées par celles contenues dans info_doc.xml.
Le format du fichier info_doc.xml est défini par le Schéma XML info_doc.xsd

ATTENTION: les documents HTML ne doivent pas contenir d'entité caractère hexadecimale comme &#x8A93; car le parseur Tidy semble ne pas les reconnaître.

1.2. Différents types d'archives

There are three types of archives, wich are treated differently:

La distinction est faite entre trois types d'archives qui sont traitées différemment :

  • une archive multilingue HTML est constituée d'un mini-site HTML dans plusieurs langues.
  • une archive multilingue non HTML contient plusieurs versions d'un document simple non HTML (par exemple un document PDF ou RTF) dans différentes langues.
  • une archive monolingue dont le contenu est libre.

Pour les deux types de documents multilingues, l'archive doit comporter à sa racine le fichier info_doc.xml puis, pour chaque langue, un répertoire portant le code à trois lettres ISO 639-2/T de cette langue. Les langues actuellement acceptées sont :
deu (allemand), eng (anglais), esp (espagnol), fra (français), hun (hongrois), ita (italien), jpn (japonais), lao (lao), msa (malais), tha (thaï), vie (vietnamien), etc.
Aucun autre fichier, à l'exception du fichier info_doc.xml,et aucun autre répertoire ne doivent être pésents à la racine de l'archive, sinon le document est considérée comme monolingue.
