Lookup 

In order to lookup a word or a concept in the dictionaries available at the Pivax server, use the lookup menu on the left in the following way:

  • Step 1: enter the word or concept you are looking for in the Word field;
  • Step 2: specify the language of the word entered or a semantic resource in the Source field;
  • Step 3: choose the desired target language or a semantic resource in the Target field. You can choose All languages to get all possible information about the word or concept;
  • Step 4: push the Go button to find translations or mappings to supported semantic resources.

Please note that some language volumes in the existing dictionaries have only very few entries. The system may be unable to find the translation to the desired language, if it does not exist in the database. The current interface has been built mainly for experimental purposes.

Online Editing 

Editing a monolingual entry (Lexie)

You have to be registered and logged to edit entries online.

the Online Editor Interface is for testers only. In order to implement the editor properly, we developed the most complete interface with all possible fields and options for specialist lexicographers. Thanks to the generic nature of the editor we will be able to develop a simplified version for non-specialist users as well.

You can add entries and then look them up through the Consultation interface.

Editing an intermediate link (Axeme)

You have to be registered and logged to edit links online.

the Online Editor Interface is for testers only. In the Pivax database architecture axemes are used as intermediate units between dictionary entries (lexies) and translation links (axies). Axemes can also serve as monolingual equivalence links connecting several dictionary entries that describe the same word or concept within one language or semantic resource.

If you want to create a monolingual equivalence link (axeme) between two entries (lexies) from different dictionary volumes for the same language, both lexies must exist in the same Pivax dictionary. If one or both entries do not exist, you must first create them using the monolingual entry editing interface. Once your two lexies are created, you can add a monoligual equivalence link (an axeme) between them.

Editing a translation link (Axie)

You have to be registered and logged to edit links online.

the Online Editor Interface is for testers only. In Pivax axies are links that connect monolingual sets of equivalent dictionary entries, represented by intermediate links (axemes). Axies represent possible translations between languages and/or semantic resources, while axemes are used withing the same language or resource.

If you want to create a translation link (axie) between two axemes, both axemes must exist in the same Pivax dictionary. If the entries do not exist, you must first create them with the previous monolingual entry editor interface. Then, once your two lexies are created, you can create a translation link (an axie) between them.

Please, note that while the pivax server and its database remain in testing status, online contributions may not be kept or integrated into the database!

Offline Editing 

You can write entries locally on your computer and then upload them to the database.

To write entries, we recommend specialized XML editing tools like XML Spy available at www.xmlspy.com. XMLSpy uses XML schemata to validate the data. It will downolad our XML schemata automatically to validate your files.

The Pivax XML schemata and examples are located at: www-clips.imag.fr/geta/services/dml/.

To check the validity of your entries online, you can use the W3C XML Schema validator: www.w3.org/XML/Schema.

Manage Contributions 

Management of your own Contributions

You have to be registered and logged to manage your contributions online.

You do so by accessing the Contribution Administration Interface. With this interface, you can see or delete your existing contributions.

Reviewing Contributions

You have to be registered and logged to acces the reviewing interface.

You have to be part of the specialist group to validate any contribution ie to integrate it definitely into the database.

You can review the contributions online by accessing the Contribution Reviewing Interface.

Access methods 

Web

The more complete way to access the Pivax server is online via a web browser. Note, that popular browsers render the same web pages in a slightly different way. In order to optimize lexical database navigation and get the most out of this site, please, use one of the following browsers:

Software that works well

  • Chimera
  • Mozilla
  • Netscape navigator 7.0 and later
  • OmniWeb 4.1
  • Opera
  • Safari

Software that poses problems

  • Amaya: The last version seems to deal with japanese characters but it is not the case for thai characters.
  • iCab: The "option" element of the HTML forms is not implemented correctly. It does not know display properly non-latin characters in the option fields. Note that this problem should not occur if you choose a language in latin characters as preferred interface language.
  • Internet Explorer: The "option" element of the HTML forms is not implemented correctly. It does not display properly non-ascii characters in the option fields. Note that this problem should not occur if you choose a language in ascii characters as preferred interface language.

Dictd protocol

The Pivax server implements the dictd protocol. You can use any dictd client, e.g OmniDictionary. A basic method could be a telnet on www.aximag.fr on port 2628. Please contact us for feedback.
