UNL Dictionary project 

This project aims at creating a multilingual lexical-semantic database serving as a dictionary of the artificial pivot language UNL (Universal Networking Language). See UNL Specifications to learn more about UNL. The Common UNL Dictionary is an open resource built for the U++ Consortium with participation of the Russian, French and Spanish UNL centers. The access will be free of charge for non commercial use (GPL and Creative Commons Open Source licenses). Our project, initiated by computational linguists, aims at being useful and open to all those who are interested in languages and applied semantics.

The lexicon of the UNL language consists of UNL Universal Words (UW). UWs are unique symbols representing complex concepts, similar to lexical senses of words. They are used in UNL semantic graphs and as pivot units for translation and linking natural language words with ontological concepts, Wordnet synsets, etc. Our dictionary includes UWs that follow additional requirements agreed upon by the members of the U++ Consortium to facilitate interoperability between different UNL-aware automatic translation systems.

The Common UNL Dictionary currently provides:

  • U++ standard compliant UWs
  • Mappings between UWs and words of the English, Russian, French and some other languages,
  • Mappings between UWs and synsets of Princeton Wordnet,
  • Mappings between UWs and Suggested Upper Merged Ontology (SUMO),
  • Information about the possible meanings of natural language words,
  • Basic grammar information for natural language words (part of speech, gender, etc.),
  • Optional links between words and dictionaries of MT systems supporting that language.

The dictionary data is periodically replicated to the resources of ETAP-3 (Russian, English) and ARIANE (French) linguistic processors to guarantee full compatibility between their UNL implementations.

Pivax server 

Pivax is a lexical database server able to host multilingual pivot type dictionaries, where individual words are linked with a pivot concept node instead of words of other languages directly. Pivax is a modification of the Papillon server software built on the same Jibiki platform.

Macrostructure

The structure of a pivax dictionary (macrostructure) is a two-level pivot structure. Each entry of each monolingual volume is linked to an intermediate technical unit called axeme, which stands for a set of all entries describing the same word of a given language or the same semantic concept. Axemes (sets of entries) from one language are linked to axemes (sets of entries) for other languages via interlingual translation links called axie. Axemes and axies are stored in separate pivot volumes.

The pivot structure greatly reduces complexity when a new language is added. There is no need to link each new entry wilth all its translations directly. The intermediate axeme level adds possibility to have several parallel dictionary volumes for the same language, e.g. British and American English or different ways of labelling the same semantic concepts.

Microstructure

The structure of the monolingual natural language entries (Microstructure) is based on the Explanatory and Combinatory Lexicography, part of the Meaning-Text Theory. This theory was created by Igor Mel'cuk and his colleagues in Moscow in the 70s gives the necessary information to go from an abstract meaning to its realisation in a given language. This theory is independent from the languages, thus the same structure is used for all the languages. It is very detailed and is suitable for human and machine usage.

Contributing

Our goal is to apply the cooperative LINUX construction paradigm to the building of a multilingual dictionary and to mutualize the efforts of creating dictionary resources. Anybody can contribute to this project by giving entire resources, or by contributing online with correcting existing data or adding new entries. The data is then publicly available. For more information about how to contribute, please read the Help section.

In order to avoid time consuming effort, we will create skeleton Pivax dictionary volumes by recuperating and integrating existing data.
