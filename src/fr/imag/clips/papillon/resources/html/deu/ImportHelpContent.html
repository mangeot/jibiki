Hilfe zum Hinzufügen von Dokumenten

Zurück zur Seite über die Verwaltung der Informations-Dokumente

Es ist möglich dem Papillon-Server ein- und mehrsprachige Dokumente jeder Art hinzuzufügen. Archiv-Dateien (TAR- or ZIP-Dateien) dürfen mehrsprachig sein, während einfache Dateien (HTML, PDF, RTF, GIF etc.) einsprachig sind.
Wenn eine GZippte Datei zum Hinzufügen geschickt wird, wird sie zuerst entpackt; die gesandte Datei wird nicht direkt hinzugefügt, sondern wird behandelt, als wäre sie in ungepackter Form geschickt worden.

1. Hinzufügen von Archiv-Dateien (.tar, .zip)

1.1. Hinzufüge-Vorgang

Das Archiv wird entpackt; sein Inhalt wird analysiert, und dann hinzugefügt. Es kann ein einzelner komprimierter Ordner sein. In diesem Fall, weist der Begriff "root" auf seinen Inhalt hin. Sie können der Archiv-Root eine XML-Datei info_doc.xml hinzufügen, die es Ihnen erspart, das Import-Formular ausfüllen zu müssen. Die info_doc.xml-Beschreibungs-Datei führt auf:

  • die Sektion, in der Sie das Dokument ablegen wollen
  • den/die Namen des/der Autoren,
  • den Titel des Dokumentes in jeder verfügbaren Sprache,
  • das Erstellungssdatum des Dokumentes (optional),
  • Referenztext im Fall einer Veröffentlichung (optional).

Wenn informationen im Formular eingegeben wurden, wird sie ignoriert und durch den Inhalt der info_doc.xml-Datei ersetzt.
Das Format von info_doc.xml ist durch das XML-Schema info_doc.xsd definiert.

Beachten Sie: Für HTML Dateien, benutzen Sie keine Hexadecimal-Zeichenentitäten wie "&#x8A93;", weil der Tidy-Parser sie nicht zu erkennen scheint.

1.2. Unterschiedliche Archiv-Typen

Es gibt drei Archivtypen, die unterschiedlich behandelt werden:

  • Ein mehrsprachiges HTML-Archiv besteht aus Mini-HTML-Seiten in verschiedenen Sprachen.
  • Ein mehrsprachiges Nicht-HTML-Archiv enthält verschiedene Versionen deselben Nicht-HTML-Dokuments (zum Beispiel ein PDF- or ein RTF-Dokument) in verschiedenen Sprachen.
  • Ein einsprachiges Archiv hat freien Inhalt (es sollte nicht dasselbe Format wie ein mehrsprachiges Archiv haben).

Ein mehrsprachiges Archiv muss an seiner Root die info_doc.xml-Datei besitzen und dann für jede Sprache einen Ordner, der den ISO 639-2/T 3 Buchstaben-Code entsprechend der Sprache trägt. Die Sprachen sind gegenwärtig: deu (Deutsch), eng (Englisch), esp (Spanisch), fra (Französisch), hun (Ungarisch), ita (Italienisch), jpn (Japanisch), lao (Lao), msa (Malayisch), tha (Thai), vie (Vietnamesisch), etc.
Keine andere Datei als info_doc.xml und kein anderer Ordner sollte auf der Archiv-Root existieren, anderenfalls wird das Archiv wie ein einsprachiges behandelt.
