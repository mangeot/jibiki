Help for Information Documents Addition

Back to Information Documents Administration Page

It is possible to add on the Papillon server monolingual or multilingual documents of any type. Archive files (TAR or ZIP files) may be multilingual, while simple files (HTML, PDF, RTF, GIF, etc.) are monlingual.
When a GZipped file is sent to be added, it is first unpacked; the file obtained is not directly added but treated exactly as if it was sent in its unpacked form.

1. Addition of archive files (.tar, .zip)

1.1. Addition Process

The archive is unpacked; its contents is analyzed, and then added. It can be a single compressed folder. In that case, the term "root" indicates its contents. You can put on the archive root an XML file named info_doc.xml wich avoids having to fill the import form. This info_doc.xml description file indicates:

  • the section where you wish to put the document,
  • the author(s) name(s),
  • the title of the document in each available language,
  • the creation date of the document (optional),
  • some reference text if it is a publication (optional).

If information have been entered in the form, it is ignored and replaced by those contained in info_doc.xml.
The format of info_doc.xml is defined by the XML Schema info_doc.xsd.

ATTENTION: for HTML files, do not use any hexadecimal character entities like &#x8A93; as the Tidy parser seems not able to recognize them.

1.2. Different archives types

There are three types of archives, wich are treated differently:

  • A multilingual HTML archive consists of a mini HTML site in several languages.
  • A multilingual non-HTML archive contains several version of a same non-HTML document (for example a PDF or a RTF document) in different languages.
  • A monolingual archive has a free content (it should not have the same format as a multilingual archive).

A multilingual archive must have on its root the info_doc.xml file and then, for each language, a folder bearing the ISO 639-2/T 3 letter code corresponding to this language.The languages currently accepted are: deu (German), eng (English), esp (Spanish), fra (French), hun (Hungarian), ita (Italian), jpn (Japanese), lao (Lao), msa (Malay), tha (Thai), vie (Vietnamese), etc.
No other file than info_doc.xml and no other folder should be on the archive root, otherwise the document will be treated as a monolingual one.
