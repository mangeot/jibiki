3.1 Preparing the Consultation

Modification of the database organisation with Dods

J'ai un peu reflechi au schema de la base de donnees et je propose de le modifier de facon a ce qu'il reflete l'organisation du namespace DML que j'ai decrit ds ma these. Pour moi, ca me permet d'etre plus confortable ds l'organisation et aussi de valider, invalider et ameliorer des idees de ma these Je pensais creer une table pour chaque fichier de DML a savoir :

1 table database ou est stockee la liste des dicos et des utilisateurs pour l'instant, il n'y aurait que Papillon mais je pense rajouter d'autres dicos pour DicoWeb (FeM, JMDict, etc.) Cles : nom de dico, nom d'utilisateur

1 table utilisateurs pour stocker le login, mot de passe, adresse mel, preferences, profils, etc. cles: nom d'utilisateur

1 table pour chaque dico ou on stocke les metadonnees et nottament un map pour chaque element de CDM. Ex pour le FeM :

  • entry =fem-entry
  • headword=ENTRY
  • pos=FRENCH_CAT

et un lien vers chaque volume cles : langues, elements CDM, metadonnees (date d'install, taille, etc...)

1 table pour chaque volume ou on stocke les entrees en XML cles : les elements CDM (headword, pos, gloss, translation, etc...)

+ tables d'historiques, de poids, etc.

Pour l'instant, il n'y a qu'une seule table dictionnaire. On peut charger des entrees de toutes les langues ds la meme table mais ca cree des pbs de visualisation, de tri, de recherche, etc. C'est pour ca que je voulais modifier le schema Dods. J'ai pu charger les entrees fra, jpn et msa que j'avais mais du coup, c'est tout en bordel. Sans compter le fait qu'il faudrait rajouter les axies...

3.2 Installing language negociation

Not so difficult, Mathieu already watched. Using the Accept_Languages variable in the header of the HTTP requests for localisation. Mybe Enhydra has already a function for this (see http://barracuda.enhydra.org).

3.3 Establishing a login session for all the users

Every user of Papillon Applicaiton has to register once and then to log on. Nothing is done for the moment. After the logon, a session has to be managed both with cookies and with URL paths. Every user will have an account on Papillon server for personal infos, settings, profiles and contributions.
